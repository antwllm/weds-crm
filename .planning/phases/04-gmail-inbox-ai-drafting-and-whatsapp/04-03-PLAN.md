    - POST /api/leads/:leadId/whatsapp/send with { message } sends WhatsApp message, stores in whatsappMessages, creates whatsapp_sent activity
    - GET /api/leads/:leadId/whatsapp returns message history ordered by createdAt asc (chat order)
    - GET /api/leads/:leadId/whatsapp/window returns { isOpen: boolean, expiresAt: string | null } based on last inbound message timestamp
    - POST send returns 400 if lead has no phone number
    - POST send returns 503 if WHATSAPP_PHONE_NUMBER_ID or WHATSAPP_ACCESS_TOKEN not configured
  </behavior>
  <action>
1. Create src/routes/api/whatsapp.ts:
   - All routes use ensureAuthenticated middleware

   POST /leads/:leadId/whatsapp/send:
   - Zod validate body: { message: string }
   - Fetch lead by id, return 404 if not found
   - Return 400 if lead has no phone number
   - Return 503 if WhatsApp config vars missing
   - Call sendWhatsAppMessage(config.WHATSAPP_PHONE_NUMBER_ID, config.WHATSAPP_ACCESS_TOKEN, lead.phone, message)
   - Insert into whatsappMessages: { leadId, waMessageId: returned ID, direction: 'outbound', body: message, status: 'sent' }
   - Insert activity: { leadId, type: 'whatsapp_sent', content: message }
   - Return 200 { status: 'sent', waMessageId }

   GET /leads/:leadId/whatsapp:
   - Query whatsappMessages where leadId, order by createdAt asc
   - Return { messages: [...] }

   GET /leads/:leadId/whatsapp/window:
   - Query last inbound whatsappMessage for this lead (direction='inbound', order by createdAt desc, limit 1)
   - If no inbound message, return { isOpen: false, expiresAt: null }
   - Calculate: isOpen = (now - lastInbound.createdAt) < 24 hours
   - expiresAt = lastInbound.createdAt + 24 hours
   - Return { isOpen, expiresAt }

2. Mount in app.ts: app.use('/api', whatsappRouter)

3. Write tests/api/whatsapp.test.ts:
  - tests/webhook-whatsapp.test.ts   - Mock sendWhatsAppMessage, mock DB
   - Test POST send success path
   - Test POST send 400 when no phone
   - Test GET history returns messages in asc order
   - Test GET window returns isOpen based on timestamp

<task type="auto" tdd="true">
  <name>Task 2: WhatsApp webhook (verification + incoming messages + SMS alert)</name>
  <files>src/routes/webhook.ts, tests/webhook-whatsapp.test.ts</files>
  <behavior>
    - GET /webhook/whatsapp with hub.mode=subscribe and correct verify_token returns hub.challenge
    - GET /webhook/whatsapp with wrong token returns 403
    - POST /webhook/whatsapp verifies X-Hub-Signature-256, responds 200 immediately, processes async
    - Incoming text message: finds lead by phone, stores whatsappMessage (direction=inbound), creates whatsapp_received activity, sends Free Mobile SMS alert
    - Incoming message with no matching lead: stores message with leadId=null or skips (log warning)
    - POST /webhook/whatsapp with invalid signature returns 403
  </behavior>
  <action>
1. In src/routes/webhook.ts, add WhatsApp webhook handlers AFTER existing Pipedrive webhook:

   GET /webhook/whatsapp:
   - Check hub.mode === 'subscribe' && hub.verify_token === config.WHATSAPP_VERIFY_TOKEN
   - If match, return 200 with hub.challenge as plain text
   - Else return 403

   POST /webhook/whatsapp:
   - If WHATSAPP_APP_SECRET configured, verify signature via verifyWebhookSignature(req.rawBody, req.headers['x-hub-signature-256'], config.WHATSAPP_APP_SECRET). Return 403 if invalid.
   - Note: Need to capture raw body for signature verification. Add express.raw() middleware for this route, or use a rawBody middleware that stores req.rawBody before JSON parsing.
   - Respond 200 immediately: res.status(200).json({ status: 'ok' })
   - Process in setImmediate (same pattern as Gmail and Pipedrive webhooks):
     - Call parseIncomingMessage(req.body)
     - If null, return (status update, not a message)
     - Find lead by phone: query leads where phone matches message.from (normalize both to compare). Use E.164 format matching.
     - If lead found:
       - Insert into whatsappMessages: { leadId, waMessageId, direction: 'inbound', body: message.text, status: 'delivered' }
       - Insert activity: { leadId, type: 'whatsapp_received', content: message.text }
       - Send Free Mobile SMS alert: "WhatsApp de {lead.name}: {message.text.slice(0, 100)}" (best-effort, same pattern as new lead alerts -- never throw)
     - If no lead found:
       - Log warning with sender phone number
       - Still store the message if possible (with leadId as null if schema allows, or skip)

2. For raw body capture: before the JSON body parser for this specific route, use express.raw({ type: 'application/json' }) or add a custom middleware that saves req.rawBody = req.body (as Buffer) before parsing. Check how the app currently handles body parsing and add rawBody capture for /webhook/whatsapp only.

   - Test POST with no matching lead logs warning
