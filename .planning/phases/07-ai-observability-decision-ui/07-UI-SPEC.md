# Phase 7 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## User Constraints (from CONTEXT.md)

### Locked Decisions
- Nouvel onglet "Decisions IA" dans la vue lead (a cote des onglets WhatsApp, Email, etc.)
- Scope onglet : WhatsApp uniquement (pas les email drafts)
- Liste chronologique (plus recent en haut), filtrable par action (reply/pass_to_human)
- Chaque entree affiche : action (badge colore vert=reply, orange=handoff), raison de la decision, message du prospect + reponse IA generee, latence + modele utilise
- Feedback depuis l'app Weds : thumbs up/down + commentaire texte optionnel sur chaque decision dans l'onglet
- Score envoye a Langfuse via API (score 1 ou 0 + comment)

### Claude's Discretion
- Design exact de l'onglet (spacing, composants shadcn a utiliser)

### Deferred Ideas (OUT OF SCOPE)
- Dashboard analytics IA integre dans l'app (au lieu de Langfuse)
- Auto-evaluation IA (un second modele qui note les reponses)

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Compact element spacing, gap between badges |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, gap between decision cards |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: 48px min-height/min-width constraint (not spacing token) for thumbs up/down buttons — WCAG 2.5.8 touch target minimum (48px > 44px requirement, stays on spacing grid)

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 400 (regular) | 1.4 |
| Heading | 16px | 600 (semibold) | 1.3 |
| Display | 20px | 600 (semibold) | 1.2 |

Note: Two weights only — 400 (regular) for body and labels, 600 (semibold) for headings/display. Label vs body distinction is handled by the 12px vs 14px size difference. The Label role (12px) is used for metadata (latence, modele, timestamp) within decision cards.

## Layout

### Tab Integration in LeadDetail

The existing `LeadDetail.tsx` has a 3-column `TabsList` grid: Notes, Emails, WhatsApp.

Phase 7 adds a 4th tab: "Decisions IA".

```
TabsList: grid-cols-4
  [Notes] [Emails] [WhatsApp] [Decisions IA]
```

The tab trigger displays `Bot` icon (16x16) + text "Decisions IA" at `text-sm font-normal`.

**Focal point:** The action badge (green/orange) in the AiDecisionCard header row is the primary visual anchor — establishes decision type at a glance.

### AiDecisionCard Structure

```
Card (border, rounded-lg, p-4, space-y-3)
  Header row (flex, items-center, justify-between):
    Left:  Badge(action) + timestamp (text-xs text-muted-foreground)
    Right: latency (Clock icon + "120ms") + model name (text-xs)

  Reason section:
    Label "Raison" (text-xs font-normal text-muted-foreground uppercase tracking-wide)
    Text (text-sm)

  Messages section (bg-muted/50 rounded-md p-3 space-y-2):
    Prospect message:
      Label "Prospect" (text-xs font-normal) + MessageSquare icon
      Text (text-sm, italic)
    AI response:
      Label "Reponse IA" (text-xs font-normal) + Bot icon
      Text (text-sm)

  Footer row (flex, items-center, justify-between, pt-2, border-t):
    Left: ThumbsUp button + ThumbsDown button (ghost variant, size sm)
           If scored: active button highlighted in blue, inactive muted
           If comment exists: small text showing comment preview
    Right: "Voir dans Langfuse" link (text-xs, ExternalLink icon)
```

### Score Feedback Interaction

1. User clicks ThumbsUp or ThumbsDown
2. Button immediately highlights (optimistic update)
3. If user wants to add a comment: a Textarea appears below the thumbs buttons (collapse/expand toggle via "Ajouter un commentaire" link)
4. Comment is submitted on blur or Enter
5. Score + comment are sent via `POST /api/ai-decisions/:id/score` and forwarded to Langfuse

## States

### Loading State
- `Skeleton` components: 3 card-shaped skeletons (h-32 each, gap-4) inside ScrollArea
- Filter bar shows disabled badge pills

### Empty State
- No decisions exist for this lead
- Centered content within ScrollArea area

### Filtered Empty State
- Decisions exist but none match current filter
- Uses muted styling within the list area

### Error State
- Failed to load decisions
- Shown inline within the tab content

### Score States
- **Unscored:** Both thumbs in `text-muted-foreground`, no fill
- **Scored positive:** ThumbsUp in `text-blue-600`, ThumbsDown stays muted
- **Scored negative:** ThumbsDown in `text-orange-600`, ThumbsUp stays muted
- **Scoring in progress:** Active button shows `Loader2` spinner (h-3 w-3)

## Responsive Behavior

This phase follows the existing LeadDetail responsive pattern:
- **Desktop (lg+):** 2-column grid `[320px sidebar] [flex-1 tabs]`. Tabs area has full width for decision cards.
- **Mobile (<lg):** Single column stack. Decision cards take full width. Filter bar scrolls horizontally if needed.
- Decision cards: full width within the tab content area. No multi-column card grid.

---

## Accessibility

- Tab trigger "Decisions IA" follows existing Tabs a11y (keyboard navigation, aria-selected)
- ThumbsUp/ThumbsDown buttons: `aria-label="Evaluer positivement"` / `aria-label="Evaluer negativement"`, `aria-pressed` reflects score state
- Filter badges: use `role="radio"` with `aria-checked` for the active filter, wrapped in `role="radiogroup"` with `aria-label="Filtrer par type de decision"`
- Langfuse external link: `target="_blank" rel="noopener noreferrer"`, `aria-label="Voir la trace dans Langfuse (nouvel onglet)"`
- Score comment Textarea: `aria-label="Commentaire sur cette decision IA"`
