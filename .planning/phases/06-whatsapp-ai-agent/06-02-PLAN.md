2. **`processWhatsAppAiResponse(lead: Lead, incomingText: string): Promise<void>`** -- main entry point:
   - Load agent config from `whatsappAgentConfig` table (limit 1). If none, log warning and return (silent pass to human, no auto-reply).
   - Check `OPENROUTER_API_KEY` exists in config. If not, log warning and return.
   - Fetch last 10 WhatsApp messages for this lead (`orderBy desc, limit 10`, then reverse for chronological order).
   - Call `buildSystemPrompt(agentConfig, lead, messages)`.
   - Call `callOpenRouter(systemPrompt, incomingText, agentConfig.model || 'anthropic/claude-sonnet-4')`.
   - Call `parseAiResponse(responseContent)` to validate JSON.
   - Check consecutive counter: if `(lead.whatsappAiConsecutiveCount || 0) >= 4` (next message would be 5th), force `pass_to_human` with reason `"5 echanges IA consecutifs sans resolution"`.
   - If `action === 'reply'`:
     - Call `sendWhatsAppMessage(config.WHATSAPP_PHONE_NUMBER_ID!, config.WHATSAPP_ACCESS_TOKEN!, lead.phone!, parsed.response)`
     - Store message: `db.insert(whatsappMessages).values({ leadId: lead.id, waMessageId, direction: 'outbound', body: parsed.response, status: 'sent', sentBy: 'ai' })`
     - Create activity: `db.insert(activities).values({ leadId: lead.id, type: 'whatsapp_sent', content: parsed.response })`
     - Increment consecutive counter: `db.update(leads).set({ whatsappAiConsecutiveCount: (lead.whatsappAiConsecutiveCount || 0) + 1 }).where(eq(leads.id, lead.id))`
   - If `action === 'pass_to_human'`:
     - Call `handleHandoff(lead, parsed.reason, messages)`

3. **`buildSystemPrompt(agentConfig, lead, messages)`** -- assemble the full prompt:
   - Determine formality: if `lead.status` is in `['nouveau', 'contacte', 'rdv', 'devis_envoye']` use `'Utilisez le vouvoiement.'`, else use `'Utilisez le tutoiement (relation etablie).'`
   - Build conversation history string: each message formatted as `[Prospect]: ...` or `[Agent IA]: ...` or `[William]: ...` based on direction and sentBy
   - Concatenate: `agentConfig.promptTemplate + '\n\n--- Base de connaissances ---\n' + (agentConfig.knowledgeBase || '(Aucune)') + '\n\n--- Contexte du lead ---\n' + lead fields + formality + '\n\n--- Historique conversation WhatsApp (10 derniers messages) ---\n' + conversationHistory`
   - Include the JSON response instruction at the end (same as in research code example)

4. **`callOpenRouter(systemPrompt, userMessage, model)`** -- make the API call:
   - Use `axios.post('https://openrouter.ai/api/v1/chat/completions', ...)` with the same header pattern as `generateDraft` in openrouter.ts
   - Headers: `Authorization: Bearer ${config.OPENROUTER_API_KEY}`, `HTTP-Referer: https://weds.fr`, `X-OpenRouter-Title: Weds CRM`, `Content-Type: application/json`
   - Body: `{ model, messages: [{ role: 'system', content: systemPrompt }, { role: 'user', content: userMessage }] }`
   - Return `response.data.choices[0].message.content`

5. **`parseAiResponse(content: string)`** -- parse and validate:
   - Strip markdown code fences: `content.replace(/```json\n?/g, '').replace(/```\n?/g, '').trim()`
   - `JSON.parse(cleaned)` then `aiResponseSchema.safeParse(parsed)`
   - If parse fails, return `{ action: 'pass_to_human' as const, response: '', reason: 'Reponse IA invalide (JSON parse error)' }`
   - Log the raw content on parse failure for debugging

6. **`handleHandoff(lead, reason, conversationHistory)`** -- pass to human:
   - Update lead: `db.update(leads).set({ whatsappAiHandoffAt: now }).where(eq(leads.id, lead.id))`
   - Anti-spam check: if `lead.whatsappAiLastAlertAt` exists and `(now - lastAlert) < 3600000` (1 hour), log info message and return without alerting
   - Update last alert timestamp: `db.update(leads).set({ whatsappAiLastAlertAt: now }).where(eq(leads.id, lead.id))`
   - Build alert message: `"Agent IA WhatsApp: passage a l'humain pour ${lead.name}\nRaison: ${reason}\n\nDerniers messages:\n${last 5 messages formatted}"`
   - Dispatch via `Promise.allSettled`:
     - Free Mobile SMS: `axios.get('https://smsapi.free-mobile.fr/sendmsg', { params: { user: config.FREE_MOBILE_USER, pass: config.FREE_MOBILE_PASS, msg: alertMsg.slice(0, 999) } })` (only if FREE_MOBILE_USER and FREE_MOBILE_PASS configured)
     - Email alert: attempt to use `getGmailClientInstance()` to send email to `config.ALLOWED_USER_EMAIL`. If Gmail client unavailable, skip (SMS is primary).
   - Log handoff event

Import all necessary modules: `axios`, `z` from 'zod', `eq, desc` from 'drizzle-orm', `getDb` from '../db/index.js', table references from '../db/schema.js', `sendWhatsAppMessage` from './whatsapp.js', `config` from '../config.js', `logger` from '../logger.js', `getGmailClientInstance` from './gmail-client-holder.js'.

Use the DI-compatible axios import pattern (direct axios, not injected) for simplicity -- the webhook context does not need test injection for phase 6.
