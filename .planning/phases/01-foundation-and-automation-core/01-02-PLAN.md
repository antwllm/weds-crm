  <action>
1. Create `src/auth/passport.ts`:
   - Configure `passport-google-oauth20` strategy using config.google.clientId, config.google.clientSecret, config.google.redirectUri
   - Request scopes: 'profile', 'email', 'https://www.googleapis.com/auth/gmail.modify', 'https://www.googleapis.com/auth/gmail.send', 'https://mail.google.com/'
   - In verify callback: check that the profile email matches config.allowedUserEmail. If not, return done(null, false). If match, store the user object with tokens (accessToken, refreshToken) in session.
   - serializeUser: store user object (id, email, displayName, accessToken, refreshToken)
   - deserializeUser: retrieve from session
   - Use `accessType: 'offline'` and `prompt: 'consent'` to ensure refresh token is always returned
   - Export `configurePassport(app)` function

2. Create `src/auth/middleware.ts`:
   - `ensureAuthenticated`: Express middleware that checks `req.isAuthenticated()`. If not, redirect to `/auth/google`. For API routes (Accept: application/json), return 401 JSON.
   - Export the middleware function

3. Create `src/routes/auth.ts`: Express Router with:
   - GET `/auth/google` -> passport.authenticate('google', { scope, accessType, prompt })
   - GET `/auth/google/callback` -> passport.authenticate('google', { failureRedirect: '/auth/failed' }), on success redirect to '/'
   - GET `/auth/failed` -> return 401 "Acces refuse - compte non autorise" (French, per INFR-02)
   - GET `/logout` -> req.logout(), destroy session, redirect to '/'

4. Create `src/routes/health.ts`: Express Router with:
   - GET `/health` -> return 200 { status: 'ok', timestamp: new Date().toISOString() }
   - No auth required (Cloud Run needs unauthenticated health checks)
