**Create src/services/pipedrive/import.ts** with:

**`importAllDeals(): Promise<{ imported: number; linked: number; skipped: number }>`**
1. Paginate through ALL deals: `GET /deals?status=all_not_deleted&start=0&limit=500`
   - Use `additional_data.pagination.more_items_in_collection` to check for more pages
   - Increment start by limit on each page
2. For each deal, call `importDeal(deal)`
3. Return summary counts
4. Log progress every 50 deals: "Import Pipedrive: {n} deals traites"

**`importDeal(deal: Record<string, unknown>): Promise<'imported' | 'linked' | 'skipped'>`**
1. Extract from deal: id, title, person_id, stage_id, custom fields (using loadFieldConfig() hash keys)
2. If person_id: fetch person via `GET /persons/${person_id}` to get email and phone
   - Email: first item in email array where primary=true
   - Phone: first item in phone array where primary=true
3. **Duplicate check**: search existing leads by email OR phone (using Drizzle `or(eq(leads.email, email), eq(leads.phone, phone))`)
   - If found: link (set pipedriveDealId, pipedrivePersonId) WITHOUT overwriting any existing CRM field values. Return 'linked'.
4. If no duplicate: insert new lead with all fields, status mapped from stage_id via `stageIdToStatus()`, source from custom field or 'pipedrive'. Return 'imported'.
5. **Import deal history** (for both linked and new leads):
   - `GET /deals/${dealId}/notes` — for each note, insert activity with type 'note_added', content = note.content, createdAt = note.add_time (preserve original date)
   - `GET /deals/${dealId}/activities` — for each Pipedrive activity (call, meeting, task, etc.), insert CRM activity with:
     - type: 'pipedrive_synced' (generic, since CRM activity types don't map 1:1)
     - content: `${activity.type}: ${activity.subject || activity.note || ''}` (e.g., "call: Appel decouverte")
     - metadata: `{ pipedrive_type: activity.type, pipedrive_id: activity.id, direction: 'import' }`
     - createdAt: activity.add_time or activity.due_date (preserve original Pipedrive date)
6. Insert pipedrive_synced activity: content 'Import depuis Pipedrive', metadata { direction: 'import' }

**Add API endpoint in src/routes/api/leads.ts:**

`POST /api/leads/:id/sync-pipedrive` — manual push of a specific lead to Pipedrive:
1. Validate id param
2. Fetch lead from DB
3. If lead not found, return 404
4. If lead already has pipedriveDealId, call `syncLeadToPipedrive(lead, 'update')` (update existing)
5. If lead has no pipedriveDealId, call `syncLeadToPipedrive(lead, 'create')` (create new)
6. This is NOT fire-and-forget — wait for the sync to complete and return the result
7. On success: return 200 `{ status: 'synced', pipedriveDealId: lead.pipedriveDealId }`
8. On error: return 500 `{ error: 'Echec de synchronisation Pipedrive' }`
Import `syncLeadToPipedrive` from `../../services/pipedrive/sync-push.js`.

**Add API function in client/src/api.ts:**
```typescript
export async function syncLeadToPipedrive(leadId: number): Promise<{ status: string; pipedriveDealId?: number }> {
  const res = await fetch(`${BASE_URL}/api/leads/${leadId}/sync-pipedrive`, { method: 'POST', credentials: 'include' });
  if (!res.ok) throw new Error('Echec synchronisation Pipedrive');
  return res.json();
}
```
