    - Test: "ignores webhook for unknown deal" — deal.change for a deal not linked to any CRM lead logs a warning and does nothing

  <action>
Create `src/services/pipedrive/sync-pull.ts` with the following exports:

**`isWithinSuppressionWindow(lastSyncAt: Date | null, windowMs?: number): boolean`**
- Returns true if lastSyncAt is within windowMs (default 5000ms) of now
- Export for testing

**`handleDealUpdate(data: Record<string, unknown>, previous: Record<string, unknown> | undefined, lead: Lead | null): Promise<void>`**
- If no lead linked to this deal, log warning and return
- Extract stage_id from data, convert via `stageIdToStatus()`
- If status changed: update lead.status, insert status_change activity (from/to), insert pipedrive_synced activity
- If custom fields changed: update lead fields (eventDate, message, source, etc.)
- Set lastSyncOrigin='pipedrive', lastSyncAt=now

**`handleDealCreated(data: Record<string, unknown>): Promise<void>`**
- Extract name, person_id, stage_id, custom fields from deal data
- If person_id exists: fetch person details via GET /persons/:id to get email/phone
- Check for existing lead by email or phone (duplicate detection — link don't duplicate)
- If existing lead found: update it with pipedriveDealId/pipedrivePersonId, don't overwrite CRM field values
- If no existing lead: create new lead with all fields, set pipedrivePersonId + pipedriveDealId
- Insert pipedrive_synced activity with metadata { direction: 'pull', action: 'create' }

**`handleDealDeleted(entityId: number, lead: Lead | null): Promise<void>`**
- If no lead linked, log and return
- Do NOT delete or change status (per user decision)
- Insert activity: type 'pipedrive_synced', content 'Deal supprime dans Pipedrive — verification requise', metadata { direction: 'pull', action: 'delete', pipedriveDealId: entityId }
- Clear pipedriveDealId on the lead (set to null) since the deal no longer exists

**`handlePersonUpdate(data: Record<string, unknown>, previous: Record<string, unknown> | undefined): Promise<void>`**
- Find lead by pipedrivePersonId
- If not found, log and return
- Extract name, email (first primary from array), phone (first primary from array) from data
- Update lead contact fields
- Set lastSyncOrigin='pipedrive', lastSyncAt=now
- Insert pipedrive_synced activity

Write tests in `tests/pipedrive/webhook.test.ts` using fixtures from `tests/pipedrive/helpers/fixtures.ts`. Mock getDb() and pipedriveApi. Test each behavior above.

  <action>
Add `POST /pipedrive` route to the existing `src/routes/webhook.ts` (alongside the existing `/gmail` route):

1. **Basic auth verification** (before processing):
   - Extract Authorization header, decode base64, compare user:password against `config.PIPEDRIVE_WEBHOOK_USER` and `config.PIPEDRIVE_WEBHOOK_PASSWORD`
   - If PIPEDRIVE_WEBHOOK_USER is not set (dev mode), skip auth check
   - If auth fails, return 401 (do NOT return 200 — Pipedrive should retry)
   - Use `node:crypto.timingSafeEqual` for password comparison to prevent timing attacks

2. **Acknowledge immediately** with `res.status(200).json({ status: 'ok' })` (same pattern as Gmail webhook)

3. **Process asynchronously via setImmediate:**
   - Parse and validate body with the Zod webhook payload schema (define inline or import)
   - If validation fails, log warning and return (don't crash)

4. **Layer 1 loop prevention:** Check `meta.change_source === 'api'` — if true, log debug "Webhook Pipedrive: changement API ignore (origine CRM)" and return

5. **Find linked lead** by pipedriveDealId (for deal events) or pipedrivePersonId (for person events)

6. **Layer 2 loop prevention:** If lead found and `lead.lastSyncOrigin === 'crm'` and `isWithinSuppressionWindow(lead.lastSyncAt)`, log debug "Webhook Pipedrive: dans la fenetre de suppression" and return

7. **Dispatch by event type** (`${meta.action}.${meta.entity}`):
   - `change.deal` -> `handleDealUpdate(data, previous, lead)`
   - `create.deal` -> `handleDealCreated(data)`
   - `delete.deal` -> `handleDealDeleted(meta.entity_id, lead)`
   - `change.person` -> `handlePersonUpdate(data, previous)`
   - Unknown -> log warning, return

8. Wrap entire setImmediate body in try/catch — log errors but never crash the server.

Import handlers from `../../services/pipedrive/sync-pull.js` and `isWithinSuppressionWindow`.
