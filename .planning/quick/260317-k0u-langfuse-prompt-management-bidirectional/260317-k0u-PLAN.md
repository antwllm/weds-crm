**1. Add schema column** in `src/db/schema.ts`:
Add `langfusePromptName` varchar(100) column to `whatsappAgentConfig` table, default `'whatsapp-agent-prompt'`. Add `langfuseSyncedAt` timestamp column (nullable). Run `bun run db:push` after schema change.

**2. Add config** in `src/config.ts`:
Add `LANGFUSE_WEBHOOK_SECRET` from `process.env.LANGFUSE_WEBHOOK_SECRET` (optional, for HMAC verification of incoming Langfuse webhooks).

**3. Create `src/services/langfuse-prompts.ts`** with three functions:

- `pushPromptToLangfuse(promptTemplate: string, promptName: string): Promise<{ version: number } | null>` -- Uses lazy-loaded `LangfuseClient` (same pattern as langfuse.ts). Calls `langfuseClient.prompt.create({ name: promptName, type: 'text', prompt: promptTemplate, labels: ['production'] })`. Returns version number on success, null if Langfuse not configured. Best-effort: log warning on failure, never throw.

- `getLangfusePrompt(promptName: string): Promise<any | null>` -- Fetches prompt via `langfuseClient.prompt.get(promptName)`. Returns the prompt object (used for trace linking). Returns null if not available. Cache the prompt object for 60 seconds to avoid repeated API calls during burst message processing.

- `pullPromptFromLangfuse(promptName: string): Promise<{ prompt: string; version: number } | null>` -- Fetches the production-labeled prompt and returns its text content and version. Used by webhook handler.

All three functions should check `langfuseEnabled` (same `!!(process.env.LANGFUSE_SECRET_KEY && process.env.LANGFUSE_PUBLIC_KEY)` check) and return null early if disabled.

**4. Modify `PUT /api/ai/whatsapp-prompt`** in `src/routes/api/ai.ts`:
After saving to DB, fire-and-forget call to `pushPromptToLangfuse(promptTemplate, result.langfusePromptName || 'whatsapp-agent-prompt')`. Update `langfuseSyncedAt` timestamp on success. Add the `langfuseSyncedAt` and `langfusePromptName` fields to the JSON response.

**5. Modify `GET /api/ai/whatsapp-prompt`** in `src/routes/api/ai.ts`:
Include `langfuseSyncedAt` and `langfusePromptName` in the response (already returned by `select()` after schema migration, just ensure the default response also includes them as null).

**6. Add webhook endpoint** in `src/routes/webhook.ts`:
Add `POST /langfuse` route (public, no auth -- same pattern as other webhooks in this file). Handler:
- Parse JSON body, check `type === 'prompt-version'` and `action === 'created'` or `action === 'updated'`.
- If `LANGFUSE_WEBHOOK_SECRET` is set, verify HMAC SHA-256 signature from `x-langfuse-signature` header against `req.rawBody` (rawBody is already saved by the express.json verify callback in app.ts).
- Check if prompt name matches a known `langfusePromptName` in `whatsapp_agent_config`.
- Only process if the incoming prompt version has the `production` label (check `prompt.labels` array in payload).
- If match: call `pullPromptFromLangfuse(promptName)` to get the actual prompt text, then update `whatsapp_agent_config.promptTemplate` and `updatedAt` in DB.
- Return 200 `{ received: true }`.
- Log all activity with French messages following project convention.

**7. Link prompts to traces** in `src/services/whatsapp-agent.ts`:
Inside `processWhatsAppAiResponse`, after getting `agentConfig`, call `getLangfusePrompt(agentConfig.langfusePromptName || 'whatsapp-agent-prompt')`. Inside the `traceAiCall` callback, after `startObservation('llm-call', ...)`, call `updateActiveObservation({ prompt: langfusePromptObj }, { asType: 'generation' })` if the prompt object is available. Import `updateActiveObservation` from the lazy-loaded `@langfuse/tracing` module -- expose it from `langfuse.ts` or import directly in whatsapp-agent.ts with a try/catch.

Note: The trace linking is best-effort. If `getLangfusePrompt` returns null (Langfuse not configured), skip the `updateActiveObservation` call silently.
