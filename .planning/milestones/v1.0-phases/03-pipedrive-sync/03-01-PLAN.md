  <name>Task 2: Retry utility with syncLog logging and test fixtures</name>
  <files>src/services/pipedrive/retry.ts, tests/pipedrive/helpers/fixtures.ts</files>
  <action>
1. **Create src/services/pipedrive/retry.ts**: Export `withRetry<T>(fn, leadId, direction, maxAttempts?)`:
   - Loop up to `maxAttempts` (default 3)
   - On success: insert into `syncLog` with status 'success', return result
   - On failure: insert into `syncLog` with status 'error' + error message
   - If all attempts exhausted: send Free Mobile SMS alert using the existing `alertNotificationFailure` from `src/services/alerts.ts` (pass failedChannel as 'pipedrive_sync'), then throw
   - Backoff: `attempt * attempt * 1000` ms (1s, 4s, 9s)
   - Use `getDb()` for syncLog inserts (same pattern as rest of codebase)
   - Direction parameter: 'push' | 'pull'
   - Log each attempt with French messages: "Tentative Pipedrive {attempt}/{max}" on retry
