  <action>
Create `src/services/pipedrive/sync-push.ts` exporting `syncLeadToPipedrive(lead: Lead, action: 'create' | 'update')`:

**For 'create' action:**
1. Load field config via `loadFieldConfig()`
2. Search for existing person by email: `GET /persons/search?term={email}&fields=email`
3. If not found, create person: `POST /persons` with `name`, `email: [{ value, primary: true }]`, `phone: [{ value, primary: true }]` (Pipedrive requires array format for email/phone)
4. Create deal: `POST /deals` with:
   - `title`: `${lead.name} (${lead.eventDate || 'date inconnue'})` (per user decision)
   - `person_id`: from step 2/3
   - `stage_id`: `statusToStageId(lead.status)`
   - `pipeline_id`: from config
   - Custom fields: `[cfg.fields.eventDate]`, `[cfg.fields.message]`, `[cfg.fields.source]`, `[cfg.fields.vcardUrl]`
5. Update lead in DB: set `pipedrivePersonId`, `pipedriveDealId`, `lastSyncOrigin: 'crm'`, `lastSyncAt: new Date()`
6. Insert activity: type 'pipedrive_synced', content 'Lead synchronise vers Pipedrive', metadata `{ direction: 'push', action: 'create', personId, dealId }`

**For 'update' action (only if lead.pipedriveDealId exists):**
1. `PUT /deals/${lead.pipedriveDealId}` with stage_id and custom fields
2. If person fields changed (email/phone/name) and lead.pipedrivePersonId exists: `PUT /persons/${lead.pipedrivePersonId}` with updated fields
3. Update lead: `lastSyncOrigin: 'crm'`, `lastSyncAt: new Date()`
4. Insert activity: type 'pipedrive_synced', metadata `{ direction: 'push', action: 'update' }`

**All API calls wrapped in `withRetry()`** from retry.ts.

Write tests in `tests/pipedrive/sync-push.test.ts`:
- Mock `pipedriveApi` (vi.mock the client module)
- Mock `getDb()` for DB operations
- Use fixtures from `tests/pipedrive/helpers/fixtures.ts`
- Test each behavior listed above

});
