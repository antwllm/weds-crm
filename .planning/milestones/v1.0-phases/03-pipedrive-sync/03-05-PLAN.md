**Fix 1: Add optimistic update to useUpdateLead in client/src/hooks/useLeads.ts**

Replicate the exact pattern already used by `useUpdateLeadStatus` (lines 57-99 in the same file):

```typescript
export function useUpdateLead() {
  const queryClient = useQueryClient();
  return useMutation({
    mutationFn: ({ id, data }: { id: number; data: UpdateLeadRequest }) =>
      apiFetch<Lead>(`/leads/${id}`, {
        method: 'PATCH',
        body: JSON.stringify(data),
      }),
    onMutate: async ({ id, data }) => {
      await queryClient.cancelQueries({ queryKey: ['leads'] });
      const previousQueries = queryClient.getQueriesData<Lead[]>({
        queryKey: ['leads'],
      });
      queryClient.setQueriesData<Lead[]>(
        { queryKey: ['leads'] },
        (old) =>
          old?.map((lead) =>
            lead.id === id ? { ...lead, ...data } as Lead : lead
          ),
      );
      return { previousQueries };
    },
    onError: (_err, _vars, context) => {
      if (context?.previousQueries) {
        for (const [queryKey, data] of context.previousQueries) {
          queryClient.setQueryData(queryKey, data);
        }
      }
    },
    onSettled: () => {
      queryClient.invalidateQueries({ queryKey: ['leads'] });
    },
  });
}
```

Key difference from current code: `onMutate` immediately updates the React Query cache with the new field values BEFORE the PATCH completes. This means `lead.name` in the component props is current for subsequent edits. `onError` rolls back on failure. `onSettled` (not `onSuccess`) triggers refetch to ensure server truth.

```typescript
function handleNameSave(part: 'first' | 'last', value: string) {
  const trimmedValue = value.trim();
  // Use the current firstName/lastName derived from lead.name
  // (now reliable thanks to optimistic update keeping lead.name fresh)
  const newFirst = part === 'first' ? trimmedValue : firstName;
  const newLast = part === 'last' ? trimmedValue : lastName;
  const fullName = [newFirst, newLast].filter(Boolean).join(' ');
  if (fullName) {
    handleSave('name', fullName);
  }
}
```

This version uses `firstName` and `lastName` constants already computed at lines 49-51 from `lead.name`. With the optimistic update in place, these values are always current. Using `filter(Boolean).join(' ')` instead of template literal avoids leading/trailing spaces when one part is empty.
